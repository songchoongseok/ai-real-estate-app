자동 커밋을 수행합니다. 아래 단계를 순서대로 실행하세요.

## 1단계: 변경사항 확인

`git status --porcelain`과 `git diff`를 실행하여 변경된 파일과 내용을 확인합니다.
변경사항이 없으면 "커밋할 변경사항이 없습니다."라고 알리고 종료합니다.

## 2단계: 변경 내용 분석 및 그룹 분류

변경된 각 파일의 diff 내용을 분석하여 관련 있는 파일끼리 그룹으로 분류합니다.

분류 기준:
- 같은 기능/모듈에 대한 변경은 하나의 그룹으로
- 관련 없는 변경은 반드시 다른 그룹으로 분리
- 설정 파일 변경(.env, config, package.json 등)은 별도 그룹으로
- 테스트 파일은 해당 소스 파일과 같은 그룹으로

## 3단계: 커밋 메시지 생성

각 그룹에 대해 conventional commits 형식으로 한국어 커밋 메시지를 생성합니다.

형식: `타입: 간결한 설명`

타입 종류:
- `feat`: 새로운 기능
- `fix`: 버그 수정
- `refactor`: 리팩토링
- `docs`: 문서 변경
- `style`: 코드 포맷팅, 세미콜론 등
- `chore`: 빌드, 설정 파일 변경
- `test`: 테스트 추가/수정

## 4단계: 그룹별 커밋 실행

각 그룹에 대해:
1. 해당 그룹의 파일만 `git add`로 스테이징
2. 생성한 커밋 메시지로 `git commit` 실행
3. 결과를 사용자에게 보고

## 5단계: 결과 요약

완료된 커밋 목록을 요약하여 보여줍니다.

$ARGUMENTS
