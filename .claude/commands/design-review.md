프론트엔드 디자인 가이드라인에 따라 현재 UI 코드를 검토하고 개선합니다.

이 스킬은 Anthropic의 frontend-design 가이드라인을 기반으로 합니다.
"generic AI slop 미학을 피하고, 의도적이고 기억에 남는 디자인"을 목표로 합니다.

## 검토 대상 파일 탐색

먼저 프로젝트의 모든 CSS/스타일 관련 파일과 컴포넌트를 찾아 읽습니다:
- `*.module.css` 파일 전체
- `index.css` (글로벌 스타일)
- 레이아웃 컴포넌트 (Layout, App)
- 주요 페이지 컴포넌트

## 디자인 검토 체크리스트

### 1. 디자인 방향성 (Design Thinking)

프로젝트의 디자인 방향이 명확한지 확인:
- [ ] **목적**: 이 인터페이스가 해결하는 문제가 디자인에 반영되어 있는가?
- [ ] **톤**: 명확한 미학적 방향이 있는가? (미니멀, 럭셔리, 에디토리얼, 브루탈리스트 등)
- [ ] **차별성**: 기억에 남는 요소가 있는가?
- [ ] **일관성**: 선택한 방향이 모든 화면에 일관되게 적용되어 있는가?

**핵심 원칙**: "명확한 개념 방향을 선택하고 정밀하게 실행하라. 대담한 맥시멀리즘과 세련된 미니멀리즘 모두 통한다 - 핵심은 의도성이지 강도가 아니다."

### 2. 타이포그래피

- [ ] **고유한 폰트**: 기본 시스템 폰트(Arial, system-ui)만 사용하고 있지 않은가?
- [ ] **폰트 페어링**: 디스플레이 폰트 + 본문 폰트가 구분되어 있는가?
- [ ] **위계 구조**: 제목, 본문, 라벨, 캡션 등 타이포그래피 위계가 명확한가?
- [ ] **크기/무게**: 폰트 크기와 weight가 체계적으로 관리되고 있는가?

**금지 항목**: Inter, Roboto, Arial 같은 과사용 폰트만 사용

**권장**: Google Fonts에서 프로젝트 성격에 맞는 특색 있는 폰트 선택
- 부동산 앱 추천: Pretendard(한국어), Noto Sans KR + 영문 디스플레이 폰트

### 3. 색상 & 테마

- [ ] **CSS 변수**: 색상이 CSS 변수로 체계적으로 관리되고 있는가?
- [ ] **주도색 + 강조색**: 지배적인 색상과 날카로운 악센트가 있는가?
- [ ] **보라 그라디언트 함정**: 흰 배경에 보라색 그라디언트 같은 클리셰를 피하고 있는가?
- [ ] **상태 색상**: 성공/경고/오류/정보 색상이 일관되게 정의되어 있는가?
- [ ] **다크/라이트**: 테마 방향이 의도적으로 선택되어 있는가?

**검증 방법**: CSS에서 하드코딩된 색상값(#3182ce 등)을 찾아 변수화 여부 확인

### 4. 모션 & 애니메이션

- [ ] **페이지 로드**: 페이지 진입 시 의도된 등장 애니메이션이 있는가?
- [ ] **Staggered reveals**: 목록/그리드 항목이 시차를 두고 나타나는가?
- [ ] **Hover 상태**: 버튼, 카드, 행에 의미 있는 hover 트랜지션이 있는가?
- [ ] **마이크로 인터랙션**: 탭 전환, 폼 제출 등에 피드백 애니메이션이 있는가?
- [ ] **CSS 우선**: 가능한 CSS-only 애니메이션 사용

**원칙**: "잘 조율된 하나의 페이지 로드 애니메이션이 산발적인 마이크로 인터랙션보다 더 감동을 준다"

### 5. 공간 구성 (Spatial Composition)

- [ ] **예상치 못한 레이아웃**: 그리드를 깨는 요소가 있는가?
- [ ] **비대칭**: 모든 것이 대칭적이고 예측 가능하지 않은가?
- [ ] **여백 활용**: 넉넉한 네거티브 스페이스 또는 의도적인 밀도가 있는가?
- [ ] **오버랩/레이어**: 요소 간 겹침이나 깊이감이 있는가?

### 6. 배경 & 시각적 디테일

- [ ] **단색 배경 함정**: 단순 솔리드 컬러만 사용하고 있지 않은가?
- [ ] **분위기 연출**: 그라디언트 메시, 노이즈 텍스처, 기하학 패턴, 레이어드 투명도 등
- [ ] **그림자**: 드라마틱한 그림자 또는 미묘한 elevation 시스템
- [ ] **장식 요소**: 보더, 디바이더, 아이콘 등 디테일 요소

### 7. 반(反) AI 슬롭 체크

다음 패턴이 발견되면 즉시 개선:
- [ ] 시스템 기본 폰트만 사용
- [ ] 흰 배경 + 파란/보라 테마의 뻔한 조합
- [ ] 모든 요소가 동일한 border-radius (6px 통일 등)
- [ ] 변화 없는 균일한 여백
- [ ] 의미 없는 회색 테두리만의 카드
- [ ] 모든 페이지가 동일한 느낌

## 수정 시 규칙

- CSS 변수를 index.css 또는 별도 variables.css에 중앙 관리
- 하드코딩된 색상값을 모두 변수로 교체
- 폰트는 Google Fonts CDN 또는 로컬 웹폰트 사용
- 애니메이션은 @keyframes + animation-delay로 구현
- 수정 후 `npm test`와 `npm run build`로 검증
- 기존 기능이 깨지지 않도록 주의

## Quick Reference: 금지 vs 권장

| 항목 | 금지 (AI 슬롭) | 권장 |
|------|----------------|------|
| 폰트 | Inter, Roboto, Arial, system-ui만 | 특색 있는 폰트 + 본문 폰트 페어링 |
| 색상 | 하드코딩, 보라 그라디언트 | CSS 변수, 주도색 + 강조색 체계 |
| 레이아웃 | 대칭적 그리드만 반복 | 비대칭, 오버랩, 그리드 브레이크 |
| 배경 | 솔리드 단색 (#f7fafc) | 미묘한 그라디언트, 텍스처, 패턴 |
| 모션 | transition만 또는 없음 | 페이지 로드 stagger, hover surprise |
| 그림자 | 동일한 box-shadow 반복 | 계층별 elevation 시스템 |
| 라운딩 | 모든 곳에 6px | 용도별 차별화 (버튼 8px, 카드 12px, 뱃지 full) |
| 테마 | 뻔한 파란+흰 | 프로젝트 성격에 맞는 의도적 팔레트 |
